# /todo

顯示目前專案的結構化待辦清單，依優先度分組，並附摘要表格。

## 資料來源

- `P:\MEMORY\memory.md`：判斷專案族群
- `P:\MEMORY\projects/{family}/status.md`：Next Actions、Blocked、待開發需求

## 優先度判斷規則

依以下條件自動分類 Next Actions 中的項目：

| 優先度 | 符號 | 條件（符合任一即適用） |
|---|---|---|
| 高 | 🔴 | User 正在等待測試、截圖/測報缺少導致無法交付、有明確版本等待合併 |
| 中 | 🟡 | 開發完成但需補文件/測報、需與 User 或後端確認後才能繼續 |
| 低 | 🟢 | 內部技術債、文件補充、無明確 deadline |

## 執行步驟

1. Read `P:\MEMORY\memory.md` — 從 workspace path 判斷專案族群（目前為 POS）。
2. Read `P:\MEMORY\projects/{family}/status.md`。
3. 取出 `Next Actions` 中所有未完成 `[ ]` 項目，跳過 `[x]` 已完成項目。
4. 依優先度規則將每個項目分入 🔴 / 🟡 / 🟢 其中一組，並從 Release 狀態表推導目標版本。
5. 取出 `Blocked` 所有項目。
6. 取出 `建檔作業待開發需求`（或類似段落）的所有項目與狀態。
7. 依以下格式輸出，今日日期取自系統：

| 🟡 中 | {票號或項目名} | {一句話說明} | {版本或 —} |
| 🟢 低 | {票號或項目名} | {一句話說明} | — |
