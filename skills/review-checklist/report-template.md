# 驗收報告格式

## Markdown 報告範本

以 Markdown 格式產出的預設驗收報告，適用於一般回覆與 spec 文件紀錄。

```markdown
## 驗收報告

### 驗收依據
- 規格文件：{spec 路徑}
- 變更範圍：{檔案清單}

### 驗收結果

#### 通過項目
| # | 檢查項目 | 狀態 | 備註 |
|---|---------|------|------|
| 1 | {檢查項目} | ✅ 通過 | {備註} |

#### 問題項目
| # | 問題描述 | 嚴重度 | 檔案:行號 | 建議修正 |
|---|---------|--------|----------|---------|
| 1 | {問題描述} | 高/中/低 | {file}:{line} | {修正建議} |

### 總結
- 通過：X 項
- 問題：X 項（高：X、中：X、低：X）
- 結論：[通過 / 有條件通過 / 不通過]
```

## 結論判定標準

| 結論 | 條件 |
|------|------|
| **通過** | 零問題，或僅有低嚴重度問題 |
| **有條件通過** | 有中嚴重度問題，但功能核心流程可運作 |
| **不通過** | 有高嚴重度問題，功能無法正常運作 |

## 問題回報格式

每個問題必須包含：

1. **問題描述**：簡潔描述問題本質
2. **嚴重度**：高 / 中 / 低
3. **檔案:行號**：精確定位問題位置
4. **建議修正**：具體的修正方向或程式碼建議

```markdown
| 1 | Request 參數轉換方法未將日期轉為時間戳，後端收到 null | 高 | feature.component.ts:85 | 使用日期轉換工具方法處理日期欄位 |
```

## 正式文件輸出

當使用者要求產出正式測報文件（非 Markdown），搭配 `docx` 或 `xlsx` skill 使用。

### .docx 驗收報告

適用於需提交給主管、PM、QA 的正式驗收文件。搭配 **docx skill** 產出 Word 文件。

**文件結構建議：**

1. **封面頁**：專案名稱、功能名稱、驗收日期、撰寫者
2. **驗收依據**：規格文件路徑、變更範圍檔案清單
3. **通過項目表格**：序號、檢查項目、狀態、備註
4. **問題項目表格**：
   - 嚴重度標色：高（紅底）、中（黃底）、低（無底色）
   - 含檔案路徑、行號、建議修正
5. **總結頁**：通過/問題統計、結論判定
6. **頁首**：專案名稱 + 功能名稱
7. **頁尾**：頁碼 + 日期

**產出指令範例：**
```
請根據這份驗收報告，使用 docx skill 產出正式 Word 驗收報告文件，
包含封面、嚴重度標色表格、頁首頁尾。
```

### .xlsx 測試矩陣

適用於追蹤多個測試案例的執行狀態。搭配 **xlsx skill** 產出 Excel 測試追蹤表。

**工作表結構建議：**

1. **Sheet 1 — 測試案例追蹤**：

| 欄位 | 說明 |
|------|------|
| 案例編號 | TC-001, TC-002, ... |
| 測試場景 | 對應 spec 的 Given-When-Then |
| 測試類型 | 快樂路徑 / 錯誤路徑 / UI 驗證 |
| 執行狀態 | 通過（綠底）/ 失敗（紅底）/ 未執行（灰底）/ 跳過（黃底） |
| 失敗原因 | 僅失敗時填寫 |
| 備註 | 補充說明 |

2. **Sheet 2 — 統計摘要**：
   - 總案例數、通過數、失敗數、通過率
   - 依測試類型分組統計
   - 建議加入通過率圓餅圖或長條圖

**產出指令範例：**
```
請根據這份 E2E 測試結果，使用 xlsx skill 產出測試矩陣 Excel，
包含案例追蹤表、狀態標色、統計摘要與通過率圖表。
```

### .docx 測試報告

適用於 E2E 測試完成後的正式測試報告文件。搭配 **docx skill** 產出。

**文件結構建議：**

1. **封面頁**：專案名稱、測試範圍、執行日期、測試人員
2. **測試環境**：瀏覽器版本、baseURL、mock 模式說明
3. **測試場景摘要表**：場景名稱、對應 spec 驗收條件、結果
4. **失敗案例詳情**：
   - 場景描述
   - 預期行為 vs 實際行為
   - 截圖（若有）
   - 失敗原因分析
5. **總結**：通過率、結論、建議後續行動
6. **附錄**：完整測試 log 或 Playwright report 連結

**產出指令範例：**
```
請根據 Playwright 測試結果，使用 docx skill 產出正式測試報告，
包含封面、測試場景摘要、失敗案例詳情與通過率統計。
```
