# 功能完整性檢查

## 欄位與畫面

- [ ] 所有規格定義的欄位是否都已實作
- [ ] 欄位型別、必填、驗證規則是否符合 spec
- [ ] 表單預設值是否正確（Input/Select: `''`、Checkbox: `false`、DatePicker: `null`）
- [ ] 表格欄位順序、名稱、格式化是否與 spec 一致
- [ ] 欄位寬度、對齊方式是否符合規格

## 畫面行為（EARS 規格合規）

- [ ] 所有 EARS 語句定義的行為是否都已實作
- [ ] 事件驅動行為（When）：按鈕點擊、表單提交等觸發邏輯
- [ ] 狀態驅動行為（While）：持續顯示/隱藏邏輯
- [ ] 條件選擇行為（If）：分支判斷邏輯
- [ ] 功能選配行為（Where）：可選功能的啟用/停用

## 使用情境（Use Cases）

- [ ] 各使用情境（新增/編輯/檢視/刪除/複製等）的進入點是否正確實作
- [ ] 情境間切換邏輯是否完整（如列表 → 編輯 → 返回列表）
- [ ] 情境前置條件是否有防呆（如編輯時資料不存在的處理）
- [ ] 不同角色/權限下的情境差異是否正確處理（按鈕隱藏、欄位唯讀等）
- [ ] 情境中的多步驟流程是否有正確的順序控制與狀態保留
- [ ] 並行情境是否有衝突處理（如多人同時編輯同一筆資料）

## 狀態管理（State）

- [ ] 元件/頁面的所有狀態是否都已實作（初始、載入中、成功、失敗、空資料、禁用）
- [ ] 狀態轉換是否正確（如提交後 idle → loading → success/error）
- [ ] 狀態間不應存在不可達或死結狀態
- [ ] 表單狀態管理：pristine/dirty/touched/invalid 各狀態下的 UI 反應是否正確
- [ ] 按鈕/操作的啟用/禁用狀態是否依規格切換
- [ ] 載入中狀態下是否禁止重複操作（防止重複提交）
- [ ] 錯誤狀態是否可恢復（如重試機制、清除錯誤後恢復操作）
- [ ] 跨元件狀態同步是否正確（如子元件修改後父元件狀態是否更新）

## 驗收條件

- [ ] 快樂路徑：所有 Given-When-Then 驗收條件是否都能滿足
- [ ] 錯誤路徑：所有錯誤場景是否都有正確處理
- [ ] 表單驗證：必填未填、格式錯誤、跨欄位驗證是否正確觸發
- [ ] 錯誤訊息：提示訊息內容是否與 spec 一致

## API 整合

- [ ] API 端點、HTTP 方法是否正確
- [ ] Request 參數轉換是否符合規格（查詢方法的參數組裝邏輯）
- [ ] Response 資料映射是否正確
- [ ] 錯誤情境處理是否完整（HTTP 錯誤、業務錯誤）

## 勾選與匯出（若適用）

- [ ] 唯一識別方法回傳正確值
- [ ] 全選邏輯使用 `.every()` 而非長度比對
- [ ] 跨分頁選取狀態保留
- [ ] 取消全選僅移除當前分頁項目
- [ ] Excel 匯出表頭與資料格式正確

## 導航與路由（若適用）

- [ ] 返回行為正確（returnUrl 傳遞與讀取）
- [ ] 離開提示正確觸發（離開確認介面實作）
- [ ] 唯讀模式下正確隱藏按鈕、禁用編輯
- [ ] 提交後快照同步
