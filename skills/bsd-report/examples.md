# BSD 測報 — 使用範例

## 基本使用

### BMPPOS（POS 專案）
```
/bsd-report BMPPOS-20
1. 送件方式、送件人員、送件單位 移至頁面最下方
2. 勾選「變更內容」後，系統自動勾選帶入「變更項目」
3. 建檔作業變更項目新增輸入說明標籤
```

### BMPPCORE（Core 專案）
```
/bsd-report BMPPCORE-280
1. I0 類型備註欄位改為選填
2. X光片/光碟片數量改為選填
3. 查詢結果清單前掃優先排序
```

## 多項目 + 混合截圖範例

```
/bsd-report BMPPCORE-280
1. I0 類型備註欄位改為選填
2. X光片/光碟片數量改為選填
3. 查詢結果清單前掃優先排序
4. 核銷清冊 PDF 前掃→後掃排序
5. 發文紀錄表 PDF 各組依輸入日期排序
6. 回覆日期欄位拒絕未來日
7. 統一回覆時間確認層防護未來日
8. 公函彈窗移除底部留白
9. 公函彈窗表格可垂直捲動
10. 查詢結果可水平捲動
```

每個項目可獨立選擇截圖模式（使用者提供 or Playwright）。

## 輸出路徑慣例

各專案 command 定義各自的輸出路徑：

| 專案 | 輸出路徑 |
|------|---------|
| BMPPOS | `K:\06.專案\06.19.保全前掃\10_正式環境PCR_IR單\2026\{TICKET}\{TICKET}_BSD.docx` |
| BMPPCORE | `K:\06.專案\06.16.理賠前掃作業\13. 正式環境PCRorIR\2026\{TICKET}\{TICKET}_BSD.docx` |

使用者可在指令最後加 `--output` 覆蓋：
```
/bsd-report BMPPCORE-280 --output D:\custom\path\BMPPCORE-280_BSD.docx
1. 項目一
```

## 項目說明文字格式

> 2026-07-22 更新：標準格式改為「修改項目 + 操作步驟」，取代舊版長段落／前後對比格式（使用者回饋過於冗長）。以下為現行標準寫法。

每個項目固定兩段：

- **修改項目**：一句話結論，不重複描述變更前狀態
  ```
  修改項目：
  儲存驗證失敗時，視窗不再自動關閉，已輸入內容不遺失
  ```
- **操作步驟**：編號條列，一步一句話，最後一步直接寫驗收結果
  ```
  操作步驟：
  1. 編輯視窗中修改「捐贈單位」保費金額，使總額與「應收保費」不符
  2. 點擊「儲存」
  3. 顯示錯誤訊息「捐贈金額總額與收據不符，請確認」，視窗維持開啟、輸入內容保留（此驗證由後端執行）
  ```
- **粗體子標題**：`修改項目：`、`操作步驟：` 兩個標籤固定用粗體段落
- **後端卡控標註**：併入操作步驟最後一條的括號內，不另起段落：
  ```
  3. ...（此驗證由後端執行，詳細規則請洽後端團隊補充）
  ```

舊格式（**已停用**，僅供對照）：
```
調整前：備註欄位為必填，未填寫時無法儲存
調整後：備註欄位改為選填，空白時可正常儲存
```

### 撰寫注意事項

所有說明文字須以**畫面角度**描述，讀者為不了解技術細節的 USER：

- ✅ 正確：「勾選『變更內容』後，下方『變更項目』區塊會自動帶入對應勾選」
- ❌ 錯誤：「changeContent checkbox 的 onChange 事件觸發 patchValue 到 changeItems FormArray」
- ✅ 正確：「點擊送出後，若『保額』欄位未填寫，畫面顯示紅字提示『請輸入保額』」
- ❌ 錯誤：「submit 時 validator 檢查 amount field 為 required，回傳 error message」

## document.xml 組裝順序

一份完整的 BSD 文件段落順序：

```
1. 標題段落（16pt 置中）
2. PCR 標籤段落（11pt 粗體）
3. 空行
4. 項目一標題（粗體）
5. 「修改項目：」標籤（粗體）+ 一句話結論
6. 「操作步驟：」標籤（粗體）+ 編號條列（1. 2. 3....）
7. 項目一截圖標籤（「畫面截圖」）
8. 項目一截圖圖片（可多張）
9. 空行
10. 項目二標題（粗體）
11. 項目二「修改項目：」+ 結論
12. 項目二「操作步驟：」+ 條列
13. 項目二截圖標籤
14. 項目二截圖圖片
... 依此類推
```
