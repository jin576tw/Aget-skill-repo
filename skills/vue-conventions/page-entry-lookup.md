# Vue 頁面入口定位法（Nuxt 2 選單型系統）

## 從 UI 名稱正查頁面

依 UI 實際 drill-in 順序逐層定位：

1. **選單名稱**：`layouts/default.vue` 的選單 `name` 屬性
2. **頁面標題**：grep `class="page_title"` 或 `class="page-title"` 找頁面實際顯示標題
3. **頁籤**：`<b-tab title="...">` 的 label
4. **鑽入頁**：跳轉時的 query param（如 `from=...`、`prePage=...`）決定行為分支

⚠️ **勿依賴 Jira ticket 標題或規格書章節名稱**——它們常與 UI 顯示不一致（規格書的功能描述 ≠ 選單路徑）。

## 從截圖反查頁面

**勿只信標題**：同模組可能有多個標題相近的頁面。判別關鍵＝把截圖上的**查詢條件欄位名稱＋動作按鈕文字**拿去 grep，命中特徵字串數最高的頁面才是入口；再用表格元件的 `url="..."` 屬性確認後端端點。

## 行為觸發路徑確認

同一頁面可能多個入口路徑，只有特定路徑帶特定 query param 才觸發目標 API。確認 bug 重現路徑時：

1. 先確認截圖頁面的實際跳轉來源（哪個列表、哪個頁籤點進來）
2. 檢查頁面程式碼中依 query param 分支的邏輯
3. 手動測試說明一律用**選單名稱、頁籤、按鈕文字**描述步驟，不用 DB ID、API endpoint、query param 等技術參數（閱讀者是非技術 user）
